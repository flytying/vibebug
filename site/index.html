Features

Everything you need to track
what breaks

A lightweight failure memory for your project. Capture, group, detect, and share — all locally.

Auto-Capture Failures

Prefix your commands with vb and failures are captured automatically. No manual logging, no extra config. Just keep coding.

$ vb npm run build
Error: Cannot find module './Button'
✓ Captured · grouped with 2 similar

Smart Grouping

Recurring failures get grouped into the same capture. Same error, same entry — no duplicate noise.

Regression Detection

VibeBug knows when a fixed failure comes back. Stop fixing the same thing twice.

Fix Tracking

Record what fixed a failure. Build a searchable history of what works so you never lose the solution again.

$ vb fix --last \
  --summary "Added null check"
✓ Marked as resolved

AI Cost Estimation

See how much debugging is costing in AI tokens and time. Know where your spend is going.

Share-Safe Summaries

Export redacted summaries that are safe to share. VibeBug strips absolute paths, terminal noise, token-like strings, and env variable values automatically.

How It Works

Three steps to
stop losing track

No config files. No accounts. No cloud. Just prefix your commands.

1

Initialize

Run vb init in your project. Creates a local .vibebug/ folder with a SQLite database.

$ cd my-project
$ vb init
✓ VibeBug initialized
2

Run Commands

Prefix your usual commands with vb. If they fail, VibeBug captures it automatically.

$ vb npm run build
ERR! Cannot find module
✓ Failure captured
3

See What Breaks

Open the local dashboard to see recurring failures, regressions, and token spends.

$ vb dash
▶ Dashboard at localhost:7600

Commands

Simple, powerful
CLI interface

Everything you need, nothing you don't. No config files, no YAML.

vb <command>

Wrap a command and auto-capture failures

vb init

Initialize VibeBug in the current project

vb dash

Open the local dashboard

vb fix --last --summary "..."

Record what fixed the latest capture

vb list

List all captured failures

vb summary

Print a compact share-safe summary

vb export --format markdown

Export a share-safe markdown report

vb ignore list

View ignored noise patterns
